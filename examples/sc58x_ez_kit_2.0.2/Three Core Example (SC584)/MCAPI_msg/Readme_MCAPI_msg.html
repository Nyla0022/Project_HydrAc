1. Description

This example is a simple illustration of a multi-core communication architecture using the MCAPI communications API.

4. Overview

The example uses MCAPI for communication between two cores on ADSP-SC5xx. The example uses connectionless message passing for sending commands and data. The example assumes that node 0 (normally core 0) is being used as a general-purpose processor (GPP) and node 1 (normally core 1) is offloading digital signal processing (DSP) tasks. The GPP acts as a master and sends commands to the slave DSP for execution. The slave executes the commands that it receives and sends responses back to the master. The simple commands are used for sending execute requests, responses and termination. In the processing stage the DSP adds some known value and sends it back to the GPP. The GPP verifies the results and then declares the test to PASS or FAIL. The example is set to build and load both core images together using multi-project references; build/load core 0 and both core 0 and core 1 will build and load. Once the debugger is started, take each code into run mode and watch for a series of inter-core messages printed to the debugger console. Both cores will terminate with "All done" console messages and then suspend at the "__lib_prog_term" pre-defined breakpoint.
